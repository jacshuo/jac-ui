# Security Policy

## Supported Versions

The first formal release line is currently supported for security updates.

| Version | Supported |
| --- | --- |
| 1.2.x | ✅ |
| < 1.2.0 | ❌ |

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues.

Instead, use one of the following private channels:

1. **GitHub Security Advisories (preferred):**
   - https://github.com/jacshuo/OnyxUI/security/advisories/new
2. **Email:**
   - jacshuo@gmail.com

When reporting, please include:

- A clear description of the vulnerability
- Steps to reproduce (proof-of-concept if possible)
- Affected versions/commit hashes
- Potential impact and attack scenario
- Any suggested remediation

## Response Timeline

As an independent maintainer, vulnerability reports are handled on a best-effort basis.

I will do my best to:

- Acknowledge receipt when possible (typically within **7 days**)
- Provide an initial assessment based on severity and available time
- Prioritize critical vulnerabilities first
- Share updates as progress is made

If you have not received a reply after **14 days**, please send a follow-up via email or GitHub Security Advisories.

## Disclosure and Remediation

- We follow coordinated disclosure and request reasonable time to investigate and patch.
- Once fixed, we may publish an advisory with impact, affected versions, and upgrade guidance.
- We credit reporters who wish to be acknowledged.

Thank you for helping keep the project and community safe.
